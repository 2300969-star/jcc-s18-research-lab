# 阶段关键包与共享商店联合可达性模型

## 研究问题

旧口径把每个缺口的抽卡概率倒数累加成“约需金币”，会把低概率目标放大成几百甚至上千金，既不是棋子价格，也不是一次真实 D 牌决策。本模型把两个问题拆开：

- **卡牌面值**：缺少棋子本身需要支付多少金币。
- **联合可达性**：在当前等级、预算和共享牌池下，能否同时拿齐这一阶段必须完成的关键包。

## 阶段关键包

路线模板从真实阶段棋盘生成三个集合：

1. 当前保血包：当前阶段上场阵容尚缺的最低必要棋子。
2. 追三里程碑：低费主 C 需要达到三星时的剩余张数。
3. 后期承接包：下一阶段棋盘相对当前棋盘新增的关键棋子。

1 至 3 级只评估当前上场所需的单张棋子，不用终局阵容硬定方向。

## 金额口径

对关键包 \(K\)，棋子 \(i\) 的费用为 \(c_i\)，目标张数为 \(q_i\)，当前持有与复制器抵扣为 \(o_i,d_i\)：

\[
G_{face}(K)=\sum_{i\in K} c_i\max(0,q_i-o_i-d_i)
\]

这个值只表示购买棋子的面值。刷新费用、抽中难度和未来经济机制均不写进 `G_face`。

## 联合抽卡模拟

每次刷新先按等级商店概率 \(\pi_{l,c}\) 抽费用层，再从该费用共享牌池按剩余份数抽具体棋子。若目标棋子出现且钱包足够则购买，并从后续共享池中移除：

\[
P(X=i\mid C=c,S_t)=\pi_{l,c}\frac{R_{i,t}}{\sum_{j:cost(j)=c}R_{j,t}}
\]

对同一状态使用固定种子的蒙特卡洛模拟，联合完成率为：

\[
\widehat P(K\text{完成})=\frac{1}{N}\sum_{n=1}^{N}\mathbf 1[K_n\text{完成}]
\]

比赛模式默认每个阶段包运行 360 次。固定状态映射固定种子，因此同一最终状态必得同一结论。

## 机制边界

只有已经确认改变商店状态的机制进入模拟：商店等级偏移、已确认剩余免费刷新、已确认同费转换、已确认复制器、已经触发且数值明确的概率修正。

开场奇遇提供的未来金币、守护者尚未建成的收益、装备、羁绊和战斗增益留在对应价值通道，不会偷渡进抽卡概率。当前无法观测对手持牌，牌库份数也仍是历史先验，结果必须展示为风险估计而非精确承诺。

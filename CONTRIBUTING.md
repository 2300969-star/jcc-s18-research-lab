# Contributing

Contributions should preserve the project's evidence-first approach.

1. Create a focused branch.
2. Add or update a fixture when correcting a recommendation failure.
3. Run `npm test` before opening a pull request.
4. Keep generated JSON and browser data in sync when changing a pipeline.
5. State whether a claim is official data, deterministic simulation, model perturbation, or real-match evidence.

Do not commit API keys, account data, private match logs, or unlicensed third-party assets.
